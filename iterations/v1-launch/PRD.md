# Good News · PRD · v1-launch

> 文件位置：`iterations/v1-launch/PRD.md`
> 配合项目根目录的 `BRIEF.md` 阅读。这份 PRD 描述首版上线要做的事情。

## 1. 项目背景
Good News（中文名暂定“今日好消息”）是一个面向中文读者的开源公益好消息聚合网站，提供网页时间线和 RSS 订阅。项目长期目标见根目录 `BRIEF.md`。

参见：[../../BRIEF.md](../../BRIEF.md)

## 2. 本次目标
首版上线后，读者可以打开网站快速看到今天或近期发生的真实好消息，并通过 RSS 持续订阅。每条好消息都应具备中文摘要、地区、主题、来源、原文链接、为什么是好消息、核实说明，避免变成鸡汤、口号式宣传或无法验证的故事。

首版每日目标收录 8-12 条精选好消息，但不为凑数降低质量；好消息少时可以少发，好消息多时也不强行卡死数量。

## 3. 用户与关键场景

### 场景 1：日常浏览好消息
- **用户**：被日常坏消息消耗、想换一个信息入口的中文读者。
- **旅程**：打开首页，看到今天或最近的好消息时间线，快速浏览多张新闻卡片。
- **结果**：读者知道每天都有具体的人和事在让世界变好，获得一点恢复感。

### 场景 2：RSS 订阅
- **用户**：习惯使用 RSS 阅读器、不想每天主动打开网站的读者。
- **旅程**：进入 RSS 订阅页，复制全部好消息 RSS，或按主题复制对应 RSS，添加到阅读器。
- **结果**：读者可以在自己的信息流里持续接收中文好消息。

### 场景 3：核实来源
- **用户**：看到某条好消息后，想确认它不是编造或宣传稿的读者。
- **旅程**：阅读卡片里的来源、原文链接和核实说明，必要时打开原文。
- **结果**：读者知道这条消息来自哪里、是否有交叉验证、是否存在需要谨慎理解的地方。

### 场景 4：按兴趣筛选
- **用户**：更关心科学医疗、公益互助、环境与动物、中国相关或世界相关好消息的读者。
- **旅程**：在首页使用地区或主题筛选，或进入主题分类页查看对应内容。
- **结果**：读者能更快找到自己关心的好消息类型。

### 场景 5：理解与参与项目
- **用户**：开发者、内容整理者，或想提供好消息线索的读者。
- **旅程**：进入关于页了解筛选标准和开源公益原则；进入提交线索页，通过 GitHub Issue 提供线索。
- **结果**：读者理解项目不是官方宣传站，也能参与补充来源和线索。

## 4. 页面结构

- `/` — 首页时间线，展示今天或近期精选好消息，是主入口。
- `/rss` — RSS 订阅页，展示全部 RSS 和主题 RSS。
- `/about` — 关于项目与筛选标准，说明项目原则、收录标准、核实方式、开源公益边界。
- `/archive` — 往期归档，按日期查看过去发布的好消息。
- `/category/[slug]` — 主题分类页，按主题查看好消息。
- `/submit` — 提交线索页，引导读者通过 GitHub Issue 提供好消息线索。

首版不单独做地区页。地区作为卡片标签和首页筛选条件存在，分为“中国”和“世界”。

## 5. 信息架构

### 首页 `/`
- **顶部导航** — 让读者快速进入核心页面。
  - 内容槽：项目名 / 今日 / 归档 / RSS / 关于 / 提交线索
- **今日摘要区** — 说明当前展示的是今天或近期的好消息。
  - 内容槽：日期 / 简短说明 / 当前收录数量 / RSS 入口
  - 默认语义：`今日`优先展示当前日期或最近 24 小时内收录的好消息；若当天内容较少，可以在下方继续展示前一天内容，但必须按日期分组，不把旧内容混成今日内容。
- **筛选区** — 让读者按地区和主题缩小信息流。
  - 内容槽：地区筛选（全部、中国、世界） / 主题筛选
- **好消息时间线** — 承载主要阅读体验。
  - 内容槽：按日期分组的新闻卡片列表 / 空状态说明 / 加载或分页入口
- **订阅提示区** — 引导读者用 RSS 持续接收更新。
  - 内容槽：一句说明 / RSS 页链接 / 全部 RSS 链接
- **项目原则简短区** — 强化“真实、具体、可验证、非宣传”的边界。
  - 内容槽：3-4 条原则 / 关于页链接
- **页脚** — 提供项目辅助入口。
  - 内容槽：开源链接 / RSS / 免责声明 / 提交线索

### 新闻卡片
- **基础信息** — 帮读者快速判断这是什么消息。
  - 内容槽：标题 / 中文摘要 / 地区标签 / 主题标签 / 发布时间或收录时间
- **来源信息** — 帮读者追溯原文。
  - 内容槽：来源名称 / 原文链接
- **解释信息** — 说明为什么收录。
  - 内容槽：为什么是好消息 / 核实说明

地区维度：`中国` / `世界`。

主题维度首版包括：`科学医疗` / `环境与动物` / `公益互助` / `公共改善` / `人物善举` / `教育文化` / `其他`。后续可以根据真实内容调整。

### RSS 页 `/rss`
- **RSS 总说明** — 告诉读者这个页面用来订阅什么。
  - 内容槽：简短说明 / RSS 使用提示
- **全部好消息 RSS** — 提供默认订阅入口。
  - 内容槽：RSS 名称 / 链接 / 复制入口
- **主题 RSS** — 让读者订阅特定主题。
  - 内容槽：主题名称 / RSS 链接 / 复制入口
- **订阅帮助** — 给不熟悉 RSS 的读者基本说明。
  - 内容槽：如何复制到 RSS 阅读器 / 推荐阅读器占位

### 关于页 `/about`
- **项目缘起** — 说明为什么做这个网站。
  - 内容槽：项目动机 / 面向读者 / 与普通新闻和正能量栏目的区别
- **好消息收录标准** — 说明什么可以被收录。
  - 内容槽：温暖积极 / 具体可验证 / 有来源 / 有建设性 / 不过度美化坏事
- **不收录什么** — 明确排除范围。
  - 内容槽：官方宣传稿 / 口号式正能量 / 鸡汤文 / 无法核实故事 / 软文 / 焦虑流量
- **官方来源处理原则** — 单独说明敏感边界。
  - 内容槽：官方发布默认不作为核心来源 / 需交叉验证 / 需有具体公共价值
- **AI 与人工确认原则** — 说明筛选和摘要如何保持谨慎。
  - 内容槽：AI 初筛 / 人工或保守复核 / 原文链接保留 / 核实说明
- **开源公益原则** — 说明免费、开源、赞助边界。
  - 内容槽：免费访问 / 开源 / 可接受赞助 / 赞助不影响选题和排序

### 归档页 `/archive`
- **日期归档** — 让读者回看过去的好消息。
  - 内容槽：日期列表 / 每日收录数量 / 当日好消息链接
- **近期回看** — 快速展示最近几天。
  - 内容槽：日期 / 卡片摘要 / 查看更多入口

### 主题分类页 `/category/[slug]`
- **主题说明** — 告诉读者当前主题范围。
  - 内容槽：主题名称 / 简短说明 / 对应 RSS 链接
- **主题时间线** — 展示该主题下的好消息。
  - 内容槽：新闻卡片列表 / 分页或更多入口

### 提交线索页 `/submit`
- **提交说明** — 告诉读者可以提供好消息线索。
  - 内容槽：提交入口 / GitHub Issue 链接 / 收录标准链接
- **需要提供的信息** — 降低无效提交。
  - 内容槽：标题 / 链接 / 来源 / 为什么你觉得这是好消息 / 是否有其他来源
- **处理原则** — 防止读者误解。
  - 内容槽：提交不代表一定收录 / 不自动发布 / 需要筛选和核实

## 6. 目标衡量

**主指标（上线后 30-90 天）：**
- 有读者持续打开网站查看好消息，首页和归档页有稳定访问。
- 有读者订阅 RSS，RSS feed 请求量或 RSS 链接点击量能被观察到。
- 提交线索入口收到真实可用的好消息线索。

**过程指标：**
- 首页访问量
- RSS 链接点击量
- RSS feed 请求量
- 原文链接点击量
- 提交线索 GitHub Issue 数量
- 每日成功收录数量
- 被拒绝或待核实线索数量

## 7. 范围

**这一版会做：**
- 首页好消息时间线
- 新闻卡片
- 地区标签和主题标签
- 首页地区和主题筛选
- RSS 订阅页
- 全部 RSS 和主题 RSS
- 关于与筛选标准页
- 往期归档页
- 主题分类页
- 提交线索页，使用 GitHub Issue 作为首版提交入口

**这一版不做：**
- 登录注册
- 评论区
- 邮件订阅
- 后台或 CMS
- 自动发布读者提交内容
- 打分式“好消息指数”
- 地区独立页面
- 移动 App
- 付费墙
- 广告排序或赞助影响排序

## 8. 风险与待解决问题

**已知常见项（请确认或调整）：**
- 新闻来源质量不稳定，需要长期维护固定来源列表和可扩展来源池。
- AI 可能误判宣传稿、软文、夸大新闻，需要人工复核或保守策略。
- 跨语言摘要可能误译，需要保留原文链接和核实说明。
- 官方来源要谨慎处理，不能把项目变成宣传搬运。
- RSS 内容粒度要平衡：既能读懂，又不能全文搬运侵权。
- GitHub Issue 提交对普通读者有门槛，但首版维护成本最低。
- 每日 8-12 条是目标，不是硬性指标；低质量内容不能为凑数进入时间线。
