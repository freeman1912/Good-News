# Follow-Up Queue

Chinese good-news leads often start as announcements, approvals, launches, or
official/corporate releases. These should not be treated as published good news
until there is visible public benefit.

Use this directory for follow-up tasks.

## Routes

- `publishable`: result is visible now and can be published after normal review.
- `follow-up`: plausible good news, but needs later evidence.
- `rejected`: not suitable for tracking or publication.

## Trackability

- `result-visible`: people, communities, patients, animals, ecosystems, or public services have already benefited.
- `follow-up-needed`: the item is promising but only announced, approved, launched, or started.
- `not-trackable`: there is no practical way to verify later.

## Verification Status

- `verified-result`: result has strong support.
- `multi-source`: more than one source supports the main claim.
- `single-source`: one accessible source supports the claim.
- `official-only`: only an official source supports the claim.
- `corporate-only`: only a company/product source supports the claim.
- `needs-follow-up`: current evidence is not enough.
- `source-inaccessible`: the original source cannot be opened or traced.
- `rejected`: outside project boundaries.

## Follow-Up Questions

Each follow-up task should ask concrete questions:

- Did the plan actually start?
- Who benefited?
- How many people, projects, communities, patients, animals, or ecosystems benefited?
- Is there a public report, follow-up article, project page, hospital page, registry, or user-side evidence?
- Did the outcome match the original claim?
- Were there signs of fraud, exaggeration, privacy harm, or pure publicity?
